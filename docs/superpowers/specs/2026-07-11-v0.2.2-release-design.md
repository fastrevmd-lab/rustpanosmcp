# v0.2.2 Dependency-Maintenance Release Design

## Goal

Publish and deploy v0.2.2 as the immutable release containing the dependency
maintenance already merged after v0.2.1. The release does not add PAN-OS tool
behavior or begin the paused multi-vsys/HA project.

## Scope

- Include the current `main` dependency graph: RMCP 2.2.0 with `sse-stream`
  0.2.4, SHA-2 0.11, and the merged patch-level Rust dependency updates.
- Include the merged GitHub Actions updates and Dependabot policy that keeps
  Rust 1.88 changes deliberate.
- Correct `PLAN.md` to state that v0.2.1 was already published and deployed.
- Bump the workspace and lockfile package versions from 0.2.1 to 0.2.2.
- Update release-facing documentation and publish a GitHub release containing
  the reproducible Linux archive and checksum.

## Exclusions

- No multi-vsys, HA, or Panorama implementation.
- No tool, authorization, inventory, token-store, mutation-state, or deployment
  configuration schema changes.
- No expansion of permitted PAN-OS XPath roots or mutation actions.

## Release gates

The release commit must pass formatting, all-feature Clippy, the locked
workspace build and test suite, documentation warnings, fuzz-target
compilation, RustSec and cargo-deny policy, packaging verification, and
byte-reproducible archive construction. Pull-request CI and final `main` CI
must both pass before creating the annotated `v0.2.2` tag.

The GitHub release must be non-draft and non-prerelease. Its archive checksum
must verify, `BUILD-INFO` must identify the tagged commit and Rust 1.88, and the
release-image workflow must succeed for the tag.

## LXC deployment

Before deployment, verify that no mutation is active or indeterminate and that
PAN-OS reports no live configuration lock. Create an the lab LXC snapshot named
`pre-v022-20260711`, preserve the v0.2.1 binary as a rollback artifact, and
install only the binary extracted from the checksummed GitHub release archive.

After restart, require:

- the installed binary hash equals the release binary hash;
- `rust-panosmcp --version` reports 0.2.2;
- systemd is enabled and active with no warning-level messages;
- the public TLS hostname and chain validate normally;
- anonymous/invalid bearer requests are rejected and the configured reader is
  authenticated;
- the read-only MCP path reaches the `panosvm` target;
- no active or indeterminate operation and no live PAN-OS config lock exists.

If any gate fails, restore the v0.2.1 binary and configuration or roll the lab LXC
back to the pre-v0.2.2 snapshot after reconciling any PAN-OS job or lock.

## Acceptance

The release is complete only when GitHub assets, tag CI, LXC binary identity,
service/TLS/authentication health, PAN-OS read access, state, lock checks, and
rollback points are all recorded without credentials or configuration
payloads.
