# v0.2.1 Maintenance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Release and deploy v0.2.1 with truthful PAN-OS lock reconciliation and default-trusted lab TLS.

**Architecture:** Keep the existing mutation lifecycle and recovery model. Make lock removal fallible, delay terminal persistence until confirmed unlock, and reuse `indeterminate` for completed-device-action/failed-unlock recovery. Use DNS-01 ACME for the private-address lab hostname and preserve digest-only bearer-token storage.

**Tech Stack:** Rust 2024, Tokio, reqwest/rustls, Axum HTTPS mocks, PAN-OS XML API, systemd, Let's Encrypt ACME, Cloudflare DNS-01, GitHub Actions and GHCR.

## Global Constraints

- v0.2.1 contains only lock-state reconciliation, lab TLS trust, credential rotation, release evidence, and the v0.3 discovery handoff.
- Never report `config_lock_held: false` unless PAN-OS confirmed lock removal or an operator completed explicit offline reconciliation.
- Never print PAN-OS keys, bearer tokens, DNS credentials, or TLS private keys.
- Preserve v0.2.0 rollback artifacts and leave the PAN-OS candidate identical to its pre-test baseline.
