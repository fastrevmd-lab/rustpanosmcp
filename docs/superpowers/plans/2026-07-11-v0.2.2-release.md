# v0.2.2 Dependency-Maintenance Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish v0.2.2 from the merged dependency-maintenance graph and deploy the exact checksummed GitHub release artifact to LXC 608.

**Architecture:** Prepare a metadata-only patch release on an isolated branch, require local and GitHub release gates, tag the final merge commit, and publish the CI-produced reproducible archive. Snapshot LXC 608, replace only its binary with verified rollback handling, validate the deployed service and PAN-OS boundary, then merge a separate evidence-only follow-up.

**Tech Stack:** Rust 1.88/Cargo, GitHub Actions and `gh`, Proxmox LXC 608 on `pve2`, systemd, native TLS, PAN-OS XML API.

## Global Constraints

- Release version is exactly `0.2.2`; never move or recreate the tag.
- Preserve Rust 1.88 as the MSRV and use `Cargo.lock` with `--locked` for gates.
- Include dependency/CI maintenance and documentation only; no Rust source, tool, schema, multi-vsys, HA, Panorama, XPath-policy, or token-policy changes.
- Publish and deploy only artifacts whose checksum and `BUILD-INFO` identify the final tagged commit.
- Preserve LXC configuration, secrets, state, certificates, and the v0.2.1 binary.
- Do not deploy while an operation is active/indeterminate or PAN-OS holds a configuration lock.
- Never print bearer tokens, PAN-OS API keys, or mutation payloads.

### Task 4: Tag and publish v0.2.2

**Files:**
- Tag: `v0.2.2`
- Publish: GitHub archive/checksum and GHCR image

**Interfaces:**
- Consumes: final `main` CI artifact
- Produces: immutable v0.2.2 release assets

- [ ] **Step 1: Require final main CI and download its artifact**

```bash
gh run watch MAIN_RUN_ID --exit-status --interval 10
rm -rf /tmp/rust-panosmcp-v0.2.2-final
mkdir -p /tmp/rust-panosmcp-v0.2.2-final
gh run download MAIN_RUN_ID --name rust-panosmcp-release-candidate \
  --dir /tmp/rust-panosmcp-v0.2.2-final
cd /tmp/rust-panosmcp-v0.2.2-final
sha256sum --check rust-panosmcp-v0.2.2-x86_64-unknown-linux-gnu.tar.gz.sha256
```

Expected: `BUILD-INFO.git_commit` equals `origin/main`.

- [ ] **Step 2: Create the tag and GitHub release**

```bash
git tag -a v0.2.2 ORIGIN_MAIN_SHA -m "rust-panosmcp v0.2.2"
git push origin v0.2.2
gh release create v0.2.2 /tmp/rust-panosmcp-v0.2.2-final/* \
  --verify-tag --title "rust-panosmcp v0.2.2" \
  --notes "Dependency-maintenance patch; no PAN-OS behavior or schema changes."
```

Expected: a non-draft, non-prerelease release contains exactly the archive and checksum.

- [ ] **Step 3: Require the tag image workflow**

```bash

```

Expected: the amd64/arm64 image, provenance, and SBOM publish successfully.
