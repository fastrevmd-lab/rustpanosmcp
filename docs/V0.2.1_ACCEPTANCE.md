# v0.2.1 acceptance evidence

Date: 2026-07-11 EDT

## Scope

v0.2.1 is limited to confirmed PAN-OS configuration-lock reconciliation,
default-trusted TLS for the lab endpoint, least-privilege writer/reviewer token
rotation, release verification, and the v0.3 Panorama discovery handoff.

## Automated regression evidence

The lock-release tests were observed RED against v0.2.0 behavior:

- successful discard persisted `config_lock_held: true`;
- simulated discard unlock failure returned false success;
- simulated post-commit unlock failure returned false success.

After the implementation, `mutation_lifecycle` passed five tests, including
successful commit/discard persistence across restart and both failed-unlock
paths persisting `indeterminate`, `config_lock_held: true`, recovery details,
and the commit job ID.

The complete local CI-equivalent run passed:

- formatting, workspace Clippy with warnings denied, locked build, 71 tests,
  documentation with warnings denied, and all fuzz targets compiled;
- RustSec audit of both lockfiles and cargo-deny license/bans/source policy;
- systemd/container packaging policy;
- two isolated release builds produced byte-identical v0.2.1 archives;
- the non-root read-only hardened container reported `rust-panosmcp 0.2.1`.

Main release-candidate CI passed all four jobs for commit `22c8cac` in
[GitHub Actions run 29134429701](https://github.com/fastrevmd-lab/rust-panosmcp/actions/runs/29134429701).

## TLS evidence

`https://rust-panosmcp.mechub.org:30031/mcp` presents:

- subject/SAN `rust-panosmcp.mechub.org`;
- issuer `Let's Encrypt YR1`;
- validity 2026-07-11 through 2026-10-09.

Default workstation verification succeeds without `--insecure` or a custom CA
argument; the unauthenticated MCP probe reaches HTTP 401. Codex token probes
initialize successfully and Claude Code reports reader, writer, and reviewer
as connected. Certbot production issuance and a staging `renew --dry-run`
succeeded through Cloudflare DNS-01. The pve2 deploy hook verifies chain,
hostname, and public-key match before an atomic `pct push`, and restores the
prior pair if the service restart fails.

## Credential evidence

`lab-change-writer` retains only device `panosvm-writer`, the exact address
XPath root, set/delete actions, and the change-set/diff/validate/commit/discard
lifecycle tools. `lab-change-reviewer` retains only get/approve change-set.
Both expire at 2026-07-11 05:07:51 EDT.

After atomic Codex and Claude configuration updates, both old credentials
returned HTTP 401 and both new credentials initialized MCP with HTTP 200.
One-time plaintext transfer files and revoked-token configuration backups were
deleted; the server store remains digest-only.

The Cloudflare DNS API token exposed during ACME diagnostics was rolled. The
replacement is active, differs from the prior credential, sees only the
`mechub.org` zone, can create and delete a temporary TXT record there, and is
denied API-token management. The prior token is rejected.

The replacement was installed in both the protected Proxmox ACME plugin and
Certbot credential with mode `0600`. A complete Certbot staging renewal
successfully created, propagated, validated, and removed the DNS-01 record for
`rust-panosmcp.mechub.org`. The one-time handoff file, invalid-token backups,
and temporary credential files were securely deleted. This closes the final
v0.2.1 release gate.
